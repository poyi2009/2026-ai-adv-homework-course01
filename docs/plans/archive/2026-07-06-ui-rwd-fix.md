# UI 響應式(RWD)修正

## User Story

身為花漾生活的使用者,在手機上瀏覽 `docs/plans/archive/2026-07-06-ui-visual-redesign-implementation.md` 落地後的新視覺時,發現該次重構只完整驗證了桌面版版型,手機版仍存在導覽列擠壓、Hero 貼邊留白錯誤、購物車/訂單清單在窄螢幕擠壓變形、後台側欄無法在手機顯示等問題。這次任務依 Pencil `.pen` 檔案中既有的手機版 frame(390px)補齊響應式版型,**不新增/移除任何功能、欄位、路由、API**。

## Context

實機以手機寬度檢視上一輪視覺重構成果後,盤點出以下落差:

- 前台 `header.ejs` 桌面版是文字導覽列,窄螢幕下直接擠壓變形,無 icon-only/漢堡選單版型。
- 首頁 Hero (`index.ejs`)手機版沿用舊有 `-mx-4` 負邊距撐滿邊界的寫法,與其餘區塊的容器間距不一致,造成手機/桌面版留白不一致。
- 購物車(`cart.ejs`)、我的訂單(`orders.ejs`)僅有桌面單列橫排版型,窄螢幕下欄位擠壓重疊。
- 後台(`admin.ejs`/`admin-sidebar.ejs`/`admin-header.ejs`)側欄固定 240px 寬度並常駐顯示,手機寬度下會直接把主內容推出畫面外;個人資訊區在窄螢幕也沒有摺疊機制。
- 後台商品/訂單管理表格(`admin/products.ejs`/`admin/orders.ejs`)僅有桌面 `<table>` 版型,窄螢幕橫向擠壓;表格文字置右對齊在窄版更不易讀。

## 實作內容

1. **前台 Header**(`views/partials/header.ejs`):桌面版維持文字導覽,新增 `md:hidden` 的 icon-only 列(購物車圖示 + 漢堡選單);漢堡選單點擊後以 `absolute top-full` 方式在標頭下方展開導覽面板(不擠壓版面),選單背景改用與頁面一致的 `bg-cream` 並置中對齊;`public/js/header-init.js` 新增 `mobileMenuToggle`/`mobileNavPanel` 的開闔邏輯,購物車數量徽章改以 `.cart-badge` class 選取(桌面/手機各一份)同步更新。
2. **首頁 Hero**(`views/pages/index.ejs`):移除 `-mx-4` 負邊距撐滿邊界寫法,改與其餘區塊採用一致的容器間距(`rounded-frame` + 全域 `px-4 md:px-8 lg:px-12` 容器),修正手機/桌面版 Hero 貼齊邊界、留白不一致的問題。
3. **購物車**(`views/pages/cart.ejs`):新增 `md:hidden` 手機版卡片版型(縮圖 + 名稱/單價/數量控制 縱向堆疊 + 刪除鍵),桌面版原有橫排列改為 `hidden md:flex` 只在桌面顯示,兩版共用同一組 Vue methods(`updateQuantity`/`confirmDelete`)。
4. **我的訂單/訂單詳情**(`views/pages/orders.ejs`、`views/pages/order-detail.ejs`):訂單清單新增手機版雙列卡片版型,取代桌面單列橫排在窄螢幕的擠壓變形。
5. **後台側欄與標頭**(`views/partials/admin-sidebar.ejs`、`views/partials/admin-header.ejs`、`views/layouts/admin.ejs`):
   - 側欄改為 `fixed md:static` + `-translate-x-full` off-canvas 抽屜,預設在手機隱藏,由標頭漢堡選單(`#admin-sidebar-toggle`)搭配 `#admin-sidebar-backdrop` 遮罩控制開闔(`views/layouts/admin.ejs` inline script)。
   - 側欄 `top-16 bottom-0`(標頭高度以下)而非 `inset-y-0`,避免蓋住 sticky 標頭導致連結被遮蔽。
   - 個人資訊區在手機改為點擊帳號圖示(`#admin-user-toggle`)展開/收合 `#admin-auth-nav` 面板,桌面版維持常駐橫排。
6. **後台商品/訂單管理表格**(`views/pages/admin/products.ejs`、`views/pages/admin/orders.ejs`):新增 `md:hidden` 手機版卡片列表,桌面 `<table>` 改為 `hidden md:block`;表格欄位文字統一由置右/置中對齊改為向左對齊(`text-right`/`text-center` → `text-left`),窄版與寬版視覺一致。
7. 修正過程中排除的兩個版面錯誤:
   - 手機選單/抽屜展開時,容器高度未隨內容增加,導致展開內容溢出但無背景色遮擋——改為 `absolute`/`fixed` 定位並補上對應背景色與 `shadow-md`。
   - 後台側欄抽屜原本從 `inset-y-0` 最頂端延伸,被 z-index 較高的 sticky header 蓋住,導致「商品管理」連結被遮蔽——改為 `top-16`(標頭高度)起算。
8. 同樣**未新增/移除任何功能、欄位、路由、API**,純響應式版型與樣式修正,不影響既有 Vue 邏輯(`v-for`/`v-model`/`data-*`)。

## 驗證

- `npm test`(vitest)全數通過,未受影響(純樣式修改,不涉及後端邏輯)。
- 實機瀏覽器於手機寬度(390px)與桌面寬度分別檢視前台 9 頁 + 後台 2 頁,確認導覽選單、購物車/訂單卡片、後台側欄抽屜皆正常開闔且不遮擋內容,由使用者於本機環境進行(執行環境的 sandbox 網路隔離無法讓 Claude 端連線到本機 dev server)。
