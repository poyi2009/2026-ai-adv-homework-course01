# UI 視覺重新規劃(夢幻粉嫩優雅風格,Pencil 畫布設計)

## User Story

身為花漾生活的品牌方,我希望網站的視覺風格能呈現「夢幻粉嫩色系 + 優雅」的質感,讓前台購物體驗與後台管理介面統一在同一套精緻的品牌語言之下,而不是像現在前台溫暖粉嫩、後台卻是純黑 console 的兩個系統。這次只做視覺層面的重新設計與 Pencil 畫布 mockup 產出,不新增任何原本沒有的功能。

## Context

「花漾生活 Flower Life」目前前台已採用粉嫩玫瑰色系(`rose-primary` #C4727F、`apricot`、`sage`、`cream`、`blush`),方向正確但仍有落差:

- **前後台視覺完全割裂**:後台(`admin.ejs` + `admin-sidebar.ejs`)是純黑底 console 風格(`bg-gray-900`),與前台暖色系毫無關聯,像兩個不同系統。
- **缺乏「優雅」的重量感**:目前色票全部落在淺色調(cream/blush/rose-light/apricot/sage),沒有較深、較有份量的強調色,標題只靠 inline `style="font-family: 'Noto Serif TC'"` 硬套,並非系統化的字體 token。
- **間距與圓角不一致**:`rounded-xl`/`2xl`/`3xl` 混用,section 之間與卡片內距偏緊,少了「留白呼吸感」。
- **無重複出現的品牌識別裝飾**:目前分隔全靠留白或普通 `<hr>`,沒有任何屬於「花」這個主題的視覺簽名。

本次任務**只做視覺層重新設計**,並在 Pencil 畫布上產出完整 mockup 給使用者檢視 ——**不修改任何程式碼、不新增任何原本沒有的功能/欄位/流程**(例如結帳頁目前沒有「付款方式選擇」UI,只在訂單詳情頁有單一顆「前往 ECPay 付款」按鈕,設計時維持這個真實流程,不虛構新步驟)。所有頁面結構、API 呼叫、表單欄位、路由都與現況一致,只調整色彩、字體、間距、圓角與裝飾語言。

使用者已確認四個關鍵方向:
1. 後台視覺**統一**為與前台同一套粉嫩優雅風格(側欄不再用純黑,但表格/Modal 結構不變)。
2. 色彩**新增**一個較深的莓紅強調色與香檳金細節色,增加「優雅」層次感,而非只調淺色。
3. Pencil 畫布**繪製全部頁面**(9 個前台頁面模板 + 2 個後台頁面模板)。
4. **響應式設計**:全部 11 個頁面皆同時繪製桌面版與手機版 2 種尺寸,完整呈現斷點下的版面收合方式。

### v2 修訂(全部 22 個 frame 完成後,依使用者回饋調整)

1. **色彩改版**:捨棄玫瑰色系,改採「薰衣草霧 Lavender Mist」紫調色系(見下方 Spec 色彩表),香檳金強調色維持不變。因所有元件與頁面皆以 Design Token 變數(`$color-rose-primary` 等)引用色彩,僅需更新 `SetVariables` 即可讓全部 22 個 frame 自動套用新色彩,無需逐頁重繪。
2. **首頁 Hero 主視覺改版**:捨棄原本置於柔霧色塊中的小圓形花束照片(使用者認為太拘謹),改為「抽象漸層」處理 —— 移除照片,Hero 背景改為柔和多色漸層網格(mesh gradient,融合 cream/rose-light/blush/rose-primary),文字置中疊加其上,並在標題上方加入花藤/花朵線稿 icon 呼應品牌簽名元素。
3. **文案精簡詩意化**:首頁 Hero 標題改為「為日常,留一束詩意」、描述精簡為「精選當季花材,手工繫成溫柔心意。」;404 頁文案改為「這裡還沒開花」(維持花卉品牌語氣,同時清楚傳達頁面不存在)。

### v3 修訂(依使用者回饋再調整)

1. **Hero 融入淺景深花束攝影照**:排版維持不變(置中文字 + CTA),但 Hero 背景改為「全幅背景照片 + 薰衣草漸層色調疊加」——實作上是同一個 frame 疊加兩層 fill:底層為淺景深花束照片(`Generate` 產生),上層為半透明(opacity 0.62)線性漸層(cream→blush→rose-primary),兼顧真實質感與品牌色彩調性、文字可讀性。
2. **精選推薦改為錯落大小網格(Asymmetric Masonry)**,與第四區塊(全部商品的齊頭式網格)區隔開來,呈現更「大氣」、有空間感的編輯感版面:
   - 桌面版:三欄式,左側 1 張大圖(本週嚴選商品,圖片下方為完整說明+價格)、中間 2 張小圖上下堆疊(深色漸層 scrim + 白字說明疊加於圖片上)、右側 1 張與左側等高的大圖(同樣 scrim 疊加說明)。
   - 手機版:改為直向堆疊版本 —— 頂部 1 張大圖(說明置於圖片下方)+ 下方 2 張並排小圖(scrim 疊加說明),精簡為 3 款商品(原 4 款,因應窄版留白需求調整呈現數量,不影響「加入購物車」等既有功能本身)。

## Spec(設計系統 Token 規劃)

### 色彩 —— 薰衣草霧 Lavender Mist(v2 改版,捨棄玫瑰色系)

| Token | Hex | 用途 |
|---|---|---|
| `--color-cream` | #F5F1FA | 全站背景基底(薰衣草霧白) |
| `--color-blush` | #E3D9F2 | Hero/淺色區塊漸層(淺丁香) |
| `--color-rose-bg` | #EFE8F7 | 卡片淺底、hover 背景(淺紫底) |
| `--color-rose-light` | #C9B3E0 | 次要強調(丁香紫) |
| `--color-rose-primary` | #B49BD1 | 主要 CTA、連結 hover(薰衣草紫) |
| `--color-rose-dark` | #8F72B3 | CTA hover 深色(深薰衣草) |
| `--color-apricot` | #D2A659 | 星等、pending 狀態(琥珀金,與 gold 區隔用) |
| `--color-sage` | #8FA888 | 已付款/成功狀態(鼠尾草綠,與紫調互補) |
| `--color-berry` | **#5C4470** | 深強調色(深梅紫):大標題重點字、後台側欄 active 態、報價數字、名言/引言裝飾 —— 拉開「優雅重量感」的關鍵色 |
| `--color-gold` | **#C9A46B**(不變) | 細線裝飾色:分隔線、卡片細邊框、icon 點綴、後台 active 邊條,是唯一跨版本維持不變的強調色 |
| `--color-text-primary/secondary/muted` | #2A2830 / #655F6E / #96909D | 文字階層(微調為偏冷色調,呼應紫色主調) |

後台中性色(取代 `bg-gray-900`):側欄改用 `--color-rose-bg`/`--color-cream` 系的淺紫底 + `text-primary` 深色字,active 項目用 `--color-berry` 文字 + `--color-gold` 左側細邊條。

> 因所有頁面與元件皆以變數引用色彩(從不寫死 hex),色彩改版僅需更新 `SetVariables` 一次即可讓 22 個 frame 全部自動套用,無需逐一重繪;僅 Style Guide 頁面中作為「參考色票」用途的展示色塊需手動更新(因該處刻意寫死 hex 以利標示色票數值)。

### 字體

- **Display(標題):`Noto Serif TC`**(已載入,目前僅 inline style 使用)—— 正式收斂為 `--font-display` token,對應 `font-display` utility class。使用時機:H1/H2、商品名稱、**價格數字**(用襯線呈現價格,像珠寶標籤而非一般電商粗體無襯線價格,是此設計的特色手法之一)、品牌 Logo、後台頁面標題。字重克制(400–500 為主,避免過粗顯得笨重)。
- **Body:`Noto Sans TC`**(已載入)—— `--font-body`,一般內文、表單、按鈕文字。
- **Label/Eyebrow:`Noto Sans TC` + 加大字距**(如 `tracking-widest text-xs uppercase`)用於小標籤(例:狀態徽章文字、表單欄位標籤、"NEW"/分類標記),不新增字體,只是既有字體的另一種排版處理。

### 間距與圓角(新增 token,統一目前混亂的用法)

- `--radius-card: 1.25rem`(卡片統一)、`--radius-frame: 1.75rem`(Hero/大容器)、`--radius-pill: 999px`(按鈕/徽章)
- Section 垂直間距由 `mb-12` 提升為 `mb-16`~`mb-20`,卡片內距由 `p-3`/`p-4` 提升為 `p-5`/`p-6`,grid `gap-4`/`gap-6` 提升為 `gap-6`/`gap-8` —— 讓版面更有呼吸感,是「優雅」的核心手法。

### 簽名視覺元素(Signature)

**一條纖細的手繪感花藤分隔線(line-art botanical divider)**,以 `gold`/`berry` 細線 + 少量花朵剪影構成,作為全站唯一重複出現、可識別的裝飾語言,用於:
- 首頁 Hero 與內容區之間的過渡
- 各 section 之間(取代純留白)
- Footer 上緣
- 空狀態(空購物車/無商品/無訂單/404)插畫語言的基底
- 後台側欄品牌區呼應前台(小尺寸版本),讓後台「看起來是同一個品牌」

其餘裝飾保持克制,不做過多花俏動效或多餘裝飾,符合「優雅」而非「甜膩」。

### 響應式設計規劃

延續專案現有 Tailwind 斷點慣例(`sm`/`md`/`lg`,現況已大量使用如 `flex-col md:flex-row`、`grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`),Pencil 畫布為每個頁面**同時繪製兩種 viewport**:

- **桌面版(Desktop)**:寬度 1440px,即目前 `lg` 以上的版面。
- **手機版(Mobile)**:寬度 390px(iPhone 12/13/14 標準寬度),對應 `sm` 以下的收合版面。

各頁面手機版收合原則(沿用現有 Tailwind 斷點行為,只做視覺重新設計,不改變收合邏輯本身):

| 頁面 | 手機版收合方式 |
|---|---|
| 風格指南 | 色票/字體/元件參考改為單欄垂直堆疊 |
| 首頁 | Hero 圖文改上下堆疊(圖在下、文字置中);精選推薦維持 2 欄;品牌故事圖上文下堆疊;全部商品網格收為單欄;好評/服務特色收為單欄 |
| 商品詳情 | 左圖右資訊改為圖在上(滿版)、資訊在下 |
| 購物車 | 品項卡維持單欄但內距縮小;總計卡由 sticky 改為列表下方一般區塊 |
| 結帳 | 兩欄改單欄堆疊,訂單摘要卡從 sticky 側欄改為表單下方區塊 |
| 登入/註冊 | 卡片滿版(留安全邊距),花藤裝飾縮小 |
| 我的訂單 | 卡片列表維持單欄,內距縮小 |
| 訂單詳情 | 訂單/收件資訊卡由並排改堆疊,商品明細表格改為可橫向捲動或簡化為卡片式列表 |
| 404 | 內容置中,插畫尺寸縮小 |
| 後台商品管理 | 側欄收合(漢堡選單或底部導覽),表格改為橫向捲動或關鍵欄位卡片化 |
| 後台訂單管理 | 同上,側欄收合、表格對應調整 |

### Pencil 畫布規劃(共 22 個 Frame:11 頁 × 桌面/手機各一)

畫布依「前台 → 後台」分區排列,每個頁面的桌面版與手機版 frame 上下相鄰放置,方便對照:

1. **風格指南 Style Guide**(新增參考頁,非既有頁面):色票、字體階層、按鈕/徽章樣式、花藤分隔線範例 —— 先立好系統再畫頁面,確保後續頁面一致。
2. **首頁 `index`**:Hero(v2 改版:移除花束照片,改為柔和多色漸層網格背景 + 置中 serif 標題「為日常,留一束詩意」+ 花藤 icon 點綴 + 膠囊 CTA)→ 花藤分隔線 → 精選推薦 4 卡 → 品牌故事(圖片單側出血,非對稱)→ 全部商品 3 欄網格(加大留白)→ 顧客好評 → 服務特色 → Footer。
3. **商品詳情 `product-detail`**:左圖(圓角 + 細金框)右資訊(serif 品名/serif 價格/柔和數量選擇器/圓潤按鈕 + 購買須知卡)。
4. **購物車 `cart`**:品項改為獨立圓角卡片(加大留白,取代目前緊湊 list)、免運進度提示 bar、總計改為「優雅收據卡」(細線分隔 + serif 金額)。
5. **結帳 `checkout`**:維持兩欄(左收件表單/右摘要卡 sticky),輸入框圓角化、focus 用 rose-primary 細線,**不新增付款方式選擇欄位**。
6. **登入/註冊 `login`**:置中卡片 + 柔霧漸層背景 + 花藤裝飾,圓潤 Tab 切換。
7. **我的訂單 `orders`**:卡片化列表,狀態膠囊改用 berry(失敗)/apricot(待付款)/sage(已付款)柔和配色。
8. **訂單詳情 `order-detail`**:付款結果 alert 精緻化、訂單/收件資訊卡、商品明細表格、「前往 ECPay 付款」CTA(維持單一既有按鈕,不新增流程)。
9. **404 頁**:花藤裝飾 + 柔和插畫感錯誤頁,文案改為「這裡還沒開花」(v2,取代原「很抱歉,找不到您要的頁面」),連結行為不變。
10. **後台商品管理 `admin-products`**:側欄改用淺色調(cream/rose-bg 底 + berry active 態 + gold 細邊條),表格/新增編輯 Modal/刪除確認 Modal 沿用前台卡片語言與新色彩系統,**結構(欄位、分頁、Modal 流程)不變**。
11. **後台訂單管理 `admin-orders`**:同上視覺系統,狀態篩選下拉、訂單列表、詳情 Modal 結構不變,狀態徽章與「我的訂單」頁一致化。

## 執行注意事項

- **Pencil 桌面 App 需為執行中狀態**:先前嘗試呼叫 `get_editor_state`/`get_guidelines` 時回傳 `transport not connected to app: desktop`,代表 Pencil 桌面應用未開啟或未連線,需先確認開啟後再呼叫 `get_editor_state(include_schema: true)` 取得 schema。
- 依 Pencil 使用規則,任何設計操作前必須先取得 `.pen` schema,必要時用 `get_guidelines` 查詢可參考的 style 預設,再用 `batch_design` 建立 frame。
- 繪製過程中定期用 `get_screenshot` 自我檢視,確認粉嫩配色不流於「甜膩扁平」、莓紅/金色細節確實帶出優雅層次感、間距是否足夠寬鬆。
- **此任務不修改任何 `.ejs`/`.js`/`.css` 原始碼**,僅在 Pencil `.pen` 畫布上產出視覺設計稿。
